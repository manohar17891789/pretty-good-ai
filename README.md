# Pretty Good AI — Voice Bot Challenge

An automated Python voice bot that calls the designated test line
(`+1-805-439-8008`), plays a realistic patient across 12 scenarios, records
and transcribes every call, and extracts concrete bugs/quality issues from
the clinic agent's behavior.

## Repo layout

```
src/                  all code
  safety.py           hard guardrail: only ALLOWED_TEST_NUMBER can be dialed
  config.py           env loading
  scenarios.py        12 patient personas/goals (scheduling, reschedule,
                       cancel, refill, hours/insurance, edge cases)
  llm.py              patient-bot conversation logic (OpenAI)
  telephony/
    server.py         Flask webhook server Twilio talks to during a call
    twiml.py          TwiML response builders (Gather/Say/Hangup)
  caller.py           places one outbound call end-to-end and saves outputs
  recording.py        downloads the Twilio recording + Whisper backup ASR
  transcript_writer.py writes per-call .md/.json transcripts
  bug_report.py       per-call bug-finding extraction + aggregate report
  run_call.py         CLI: run a single scenario
  run_all.py          CLI: run every scenario, then build the bug report
  simulate.py         text-only dry run (no telephony) for fast iteration
calls/
  recordings/         downloaded call audio (mp3)
  transcripts/        per-call transcripts (.md + .json)
docs/
  architecture.md
  bug_report.md        generated by run_all.py / src.bug_report
  changelog.md          iteration log
  loom_walkthrough_plan.md
  debug_walkthrough_plan.md
tests/                 smoke tests (no network/telephony required)
```

## Setup

1. Python 3.11+, a Twilio account with a phone number, and an OpenAI API key.
2. `pip install -r requirements.txt`
3. `cp .env.example .env` and fill in:
   - `TWILIO_ACCOUNT_SID`, `TWILIO_AUTH_TOKEN`, `TWILIO_CALLER_NUMBER` (the
     one Twilio number used for every call)
   - `OPENAI_API_KEY`
   - `PUBLIC_BASE_URL` — see step 4
4. Twilio needs a public HTTPS URL to send webhooks to. In a second
   terminal: `ngrok http 5000`, then copy the `https://...ngrok-free.app`
   URL into `PUBLIC_BASE_URL` in `.env`.

## Running

Start the webhook server (keep this running in its own terminal):

```
python -m src.telephony.server
```

List available scenarios:

```
python -m src.run_call --list
```

Run a single scenario call:

```
python -m src.run_call schedule_simple
```

Run every scenario (10+ calls) and build the aggregated bug report — this
is the one main command for a full submission run:

```
python -m src.run_all
```

Each call produces:
- `calls/recordings/<scenario>_<timestamp>.mp3`
- `calls/transcripts/<scenario>_<timestamp>.md` and `.json`
- `docs/findings/<scenario>.json` (per-call bug findings)

`python -m src.run_all` finishes by regenerating `docs/bug_report.md` from
every finding produced so far.

### Fast iteration without spending telephony minutes

```
python -m src.simulate schedule_simple
```

Runs the same patient persona against a mock LLM "clinic agent" entirely
in text — useful for tuning a scenario's persona/goal before spending real
call minutes on it.

### Tests

```
python -m pytest
```

Covers the safety guardrail, scenario definitions, and TwiML generation —
no network or credentials required.

## Safety

`src/safety.py` hardcodes the only number this bot will ever call
(`+1-805-439-8008`). `src/caller.py` checks every outbound call against it
before touching the Twilio API, independent of anything in `.env`.

## Cost

Twilio outbound US calls are billed per-minute (~$0.013/min) plus
per-minute recording (~$0.0025/min); 12 short scripted calls (~1–2 min
each) plus OpenAI usage (gpt-4o-mini for conversation/bug-analysis,
Whisper for backup transcription) comes out to a few dollars total, well
under the $20 budget.
